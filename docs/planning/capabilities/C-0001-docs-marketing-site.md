# Documentation & marketing site

## Summary

- The template ships a ready-to-run documentation and marketing website, so a project generated from it has a public front door and a docs/API reference on day one with no framework setup. ^summary
- Realized as a single Astro site with Starlight for docs, TypeDoc-generated API reference, and a co-located marketing landing page — kept split-ready per [[D-0001-docs-marketing-site-tooling]].

## Statement

A project generated from this template can publish a website that serves two
audiences from one build: a marketing/landing surface that explains what the
project is, and a documentation surface — guides plus autogenerated API
reference — for people using it. The ability is provided out of the box: the
site builds, serves locally, and deploys as static output without the author
first wiring up a framework, search, theming, or an API-doc pipeline.

## What it provides

- A pre-configured Astro + Starlight site with working navigation, sidebar, dark mode, and built-in (Pagefind) search.
- A marketing landing page co-located in the same project (Astro pages outside the docs route).
- Autogenerated API reference from TSDoc comments (TypeDoc → Markdown → Starlight sidebar).
- Local dev and static build/deploy scripts wired into the template's toolchain.
- Docs content behind a clean `src/content/docs` boundary, so a later split into a standalone docs site stays cheap.

## Hook points

- Branding/theming: Starlight config (logo, colors, social links) and Astro layout components — one styling surface that covers the docs, marketing pages, and the autogenerated API reference alike.
- Content: authors add MDX under `src/content/docs`; marketing pages under `src/pages`.
- API surface: TypeDoc entry points / tsconfig control what gets documented.

## Underlying implementation

- Planned per [[D-0001-docs-marketing-site-tooling]]: Astro, `@astrojs/starlight`, `starlight-typedoc`, and TypeDoc. Not yet built — status `open/planned`.

## Notes

- "Split-ready" is a design constraint on the single site, not a second site today. See the decision's Out-of-scope and Open-questions sections.
